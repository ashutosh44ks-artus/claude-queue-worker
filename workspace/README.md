## Supabase RLS setup for Products

If `/test` shows `new row violates row-level security policy for table "Products"`, run this in the Supabase SQL Editor:

```sql
alter table public."Products" enable row level security;

grant usage on schema public to anon, authenticated;
grant select, insert on table public."Products" to anon, authenticated;

create policy "Allow public read Products"
on public."Products"
for select
to anon, authenticated
using (true);

create policy "Allow public insert Products"
on public."Products"
for insert
to anon, authenticated
with check (true);
```

For template testing this is fine. For production, tighten policies (for example, only `authenticated` users or ownership checks).
